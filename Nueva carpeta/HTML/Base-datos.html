Base de Datos

Una base de datos es un conjunto organizado de información que se almacena para poder ser utilizada de manera rápida y eficiente. Su objetivo principal es permitir guardar datos, buscarlos, actualizarlos o eliminarlos sin perder orden ni consistencia. Para administrar y trabajar con una base de datos se utiliza un Sistema Gestor de Bases de Datos (SGBD), que es un programa especializado como MySQL, PostgreSQL, SQL Server, Oracle o SQLite. Estos sistemas permiten crear tablas, definir relaciones, ejecutar consultas y mantener la integridad de la información. En este trabajo lo que se ve es cómo entender y reconocer una base de datos que no está normalizada, o sea, que tiene un montón de datos repetidos y mezclados en las mismas celdas. Arranca con una tabla que ya viene con problemas: nombres repetidos, teléfonos duplicados y hasta listas de productos y cantidades metidas en un solo casillero. En las respuestas vos vas señalando esas cosas y explicás que eso complica todo, porque si querés buscar, cambiar o analizar algo, se vuelve un lío y hay riesgo de errores o inconsistencias. Después armas tu propia tabla sin normalizar, que en este caso es de un kiosco, con varios productos en la misma celda, cantidades y precios puestos de a varios, y repitiendo los mismos clientes en distintas filas. Esa parte está buena porque justamente genera los problemas que después hay que detectar: redundancia, dificultad para actualizar datos y errores si se tipea distinto el mismo nombre. Más adelante, identificás los problemas y explicás por qué sería mejor normalizar, separando los datos en distintas tablas. En la última parte ya mostrás cómo lo harías: una tabla de clientes con un ID único para cada uno, y otra tabla de ventas donde solo se guarda el ID del cliente, un solo producto por fila y sus datos. Ahí ya se ve cómo la info queda más ordenada, sin repeticiones innecesarias, y más fácil de mantener. En general, el trabajo pasa de detectar los errores y las complicaciones de una base mal armada, a proponer una estructura más limpia que evita que se repitan datos y hace que todo sea más simple de manejar.

SQL

A lo largo del curso, trabajamos en el manejo de bases de datos utilizando SQL, con el objetivo de comprender cómo organizar y gestionar información de manera eficiente. Comenzamos aprendiendo a crear tablas para almacenar distintos tipos de registros, como el stock de una tienda de electrodomésticos, incluyendo campos para el nombre del producto, precio, cantidad disponible y proveedor. También exploramos comandos esenciales del lenguaje SQL, como CREATE TABLE, INSERT INTO, UPDATE y SELECT, que nos permitieron estructurar, registrar y consultar datos con precisión. A medida que avanzamos, fuimos aplicando relaciones entre tablas para optimizar la organización de la información y asegurar la integridad de los datos. Cada actividad realizada nos ayudó a afianzar el uso de SQL como herramienta clave para la gestión de datos en entornos reales.

Modelo Entidad–Relación

Hice un trabajo sobre el modelo entidad-relación y acá te lo explico como si tuviera que contárselo a alguien en el cole, sin palabras raras. Al principio todo me parecía un lío, pero lo que hice fue armar un ejemplo de biblioteca escolar para entenderlo mejor. Creé 3 tablas principales: Estudiantes, Libros y Préstamos, cada una con sus datos, por ejemplo Id_Alumno (pk), Nombre, Apellido, Curso; en Libros Id_Libro (pk), Título, Autor; y en Préstamos puse FechaPrestamo, FechaDevolucion y las fk que apuntan a Id_Alumno e Id_Libro.

La idea básica es que con las claves (pk y fk) se sabe quién tiene cada libro y no se repiten los datos, si no pusieras esto todo estaría desordenado. También vimos la cardinalidad, que es como contar cuántas cosas se relacionan, por ejemplo: un alumno puede tener muchos préstamos (1:N) y un libro puede aparecer en muchos préstamos (N:M).

Lo dibujé en draw.io y después lo pasé a un esquema más simple. Me ayudó un montón porque verlo gráfico hace que se entienda. Es como un plano para ordenar la info antes de guardarla, y aunque al principio daba vueltas, haciendo el ejemplo de la biblioteca lo terminé entendiendo.
